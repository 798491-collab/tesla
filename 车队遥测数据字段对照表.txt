# Tesla Fleet Telemetry 车队遥测数据字段对照表

> 本文档整理了 Tesla Fleet Telemetry（车队遥测）所有可用字段，并标注了在项目中的映射状态。
> 更新日期：2026-06-03

---

## 字段映射状态说明

| 标记 | 含义 |
|------|------|
| ✅ | 已在 Protobuf + JSON + REST 三条路径完整映射 |
| ⚠️ | 部分路径已映射，部分路径缺失 |
| ❌ | 未映射（仅通过 default 分支以 snake_case 存储） |

## 数据管道架构说明

### 两条数据路径

| 路径 | 数据源 | 推送方式 | 前端事件 |
|------|--------|---------|---------|
| **车队遥测** | Tesla → 服务器（mTLS） | 服务器 → 前端（WebSocket） | `realtime_update` / `state_update` / `media_state` |
| **REST API 轮询** | 服务器 → Tesla REST API | 服务器 → 前端（WebSocket） | `vehicle_state` / `realtime_update` / `state_update` / `online_state` |

### 两种路径的自动切换

- 当车队遥测连接正常时，使用遥测数据（实时性高，字段丰富）
- 当遥测连接断开时，自动降级为 REST API 轮询（字段相对少，但有基础数据）
- 前端通过 `vehicleStore.source` 判断当前数据来源

### 关键处理差异

| 处理项 | 遥测路径 | REST API 路径 |
|--------|---------|--------------|
| 坐标系 | WGS-84 → GCJ-02 转换 ✅ | WGS-84 → GCJ-02 转换 ✅ |
| 速度单位 | km/h（直接使用） | mph → km/h 转换 |
| 里程单位 | 英里 → 公里 转换 | 英里 → 公里 转换 |
| Redis 存储 | 增量合并 | 整体覆盖 |
| 未映射字段 | default 分支自动 snake_case 存储 | 丢弃 |
